Clumped Box Vision System

A project as part of the mechatronics systems integration course, was to implement an improvement in the conveyor system used during our labs. Our improvement aimed to fix the edge case where boxes would get clumped together. This happened for various reasons such as a box getting stuck during a previous section of the conveyor line and unstuck due to another box hitting it, or even due to user error at spacing the boxes apart correctly. This is an issue, as when boxes are stuck together or too close, the pre-existing system for sorting the boxes would divert all of the boxes regardless of its type based on the first one. This is due to the nature of the diverter system that is part of the pre-existing system as seen below.

Design Parameters

There were a few design parameters that were set for this project. These are as follows:
  1. If a clump (2 boxes in close proximity) is detected, the system will divert the boxes, and let it go to an assumed accumulator, where an operator can help deal with the issue in a timely manner.
  2. System will be able to detect two boxes with a maximum distance of one box length apart.
  3. System will be able to detect groups of two objects, regardless of orientation (limited to a range of box sizes).
  4. System should be able to flag, and relay clumped box information to the operator.
  5. System should be able to retain all functionality from the reference case. In other words, the system should be an addition to the pre-existing conveyor system.

Image Detection

To determine whether a clumped case was present, image detection was implemented. The existing conveyor architecture already included a camera and vision system to allow for the sorting of the boxes. Thus, to solve this problem, an additional script was implemented. By taking advantage of simple image detection techniques, clumped cases could be determined. This involved cropping the captured video so that all unnecessary visual noise could be removed. This is possible do to the controlled environment. A HSV (Hue Saturation and Value) mask can then be applied to this cropped frame. This allows for the filtering and removal of noise. Then using the built in contour detection within the OpenCV library, the boxes can be detected. Further filtering can be done if necessary by using the area of the detected polygons and removing false detections. Human Machine Interface (HMI)

One of the required design parameters for this project was to be able to flag and relay this information to an operator. This is where the HMI comes in. A simple graphical user interface (GUI) was created using Ignition 8 Designer. MQTT, an internet of things (IOT) messaging protocal was used to retrieve the data from the vision system and thus allow for the flaging and relaying of relevant information. Result

After implementing this new add-on to the existing conveyor system, it was able to work perfectly with it detecting and diverting all cases where the boxes were too close together (clumped cases). As well, the HMI was able to consistently flag and relay the correct information.
